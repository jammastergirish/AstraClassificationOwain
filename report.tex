Can a large language model articulate a classification rule it learned from a few in-context examples? And are there 
rules the model can \emph{apply} accurately but not \emph{describe}? For each rule, I generated balanced (50/50) labelled short nonsense
sentences, then ran two tasks: \emph{use it} (classify a held-out
item from a single-token, no-reasoning prompt) and \emph{state it} (articulate the
rule). Across seventeen rules on Claude Opus~4.8, almost every rule the model
classified well, it also articulated well. The exceptions were two rules whose feature was the
first letter inside a word (``first letter is a vowel''; ``first letter is a or e''),
which it classified at 88--100\% but stated as a meaning-based \emph{substitute}
(``the first word is a function word'').

A counterfactual test on minimal pairs that
cleaved the intended rule apart from the substitute---e.g.\ a novel vowel-initial content
word such as ``orange'', for which the intended rule says \textsf{True} but the substitute
says \textsf{False}---showed the substitute, not the intended rule, governed behaviour
($\sim$63\% of decisions). The model still knew the
letter concept perfectly in isolation (100\%). Thus the apparent ``articulation
failure'' was \emph{underdetermination}: the model learned a different rule to the one I
intended and reported it faithfully. The substitute was controllable: decorrelating the
training examples (95\%) or letting the model reason before answering (91\%) recovered
the intended rule.

Finally, I repeated the experiment on (an open-weights) Gemma~4 31B, where a linear probe read \emph{both} the intended rule (86\%) and the
substitute (96\%) from the residual stream at what appeared to be the decision point.

\section{Introduction}
A language model can learn a pattern from a handful of labelled examples placed in its
context and then continue the pattern on new items, with no finetuning or retraining; this is \emph{in-context
learning}. Following a \href{https://docs.google.com/document/d/15pnkvBciYO07wl3yjm1LO3_64soHDxsDXTMlD7ujuzI/edit?tab=t.0\#heading=h.wgm99hjsgzbj}{\textcolor{linkcol}{brief from Owain Evans}}, I ran two tests on Claude Opus~4.8. The first asked whether the model could \emph{use} a rule: given labelled examples,
did it label new items correctly? The second asked whether it could \emph{state} that rule
in words. The question I was really after combined the two: \textbf{were there rules a
model could reliably use but not describe?}. This is effectively my \emph{money
quadrant}---a rule classified correctly but not articulated. If such rules existed, a model
could be governed by a rule it could not articulate.

My primary finding was that the naive version of the money quadrant was an illusion.
Two rules looked like clean instances---high classification accuracy, low articulation
accuracy---but a counterfactual test showed the model never learned the rule I
\emph{intended}; it had learned an easier rule that agreed on the training data and reported
\emph{that} rule honestly. Properly measured, the money quadrant was empty: every rule the
model genuinely learned, it could also describe. The interesting structure was \emph{why}.

\section{Method}
\label{sec:methods}
\paragraph{Tasks.} For each rule, I wrote a \emph{generator} that invented short nonsense
sentences (e.g.\ ``kitten loud bright house'') and a \emph{labeller} that marked each as 
as \textsf{True}/\textsf{False}. Every batch was balanced. Table~\ref{tab:inputs} gives representative inputs for the
load-bearing rules.

\paragraph{Step 1 --- use it (classify).} I showed the model $\sim$50 labelled examples then one new
sentence which required a (\textsf{True}/\textsf{False}) response, with reasoning
turned off and \texttt{max\_tokens} capped. The idea was that the model could not reason in the visible
response. The instruction did not hint at the rule, so it could not leak the answer into Step~2.

\paragraph{Step 2 --- state it (articulate).} I showed the same labelled examples and asked the model to articulate therule in one sentence without reasoning (a single shot) and
with chain-of-thought (CoT) allowed.

\paragraph{Step 3 --- the decisive test (counterfactual).} For each rule I constructed
\emph{minimal-pair} inputs on which the intended rule and a simpler candidate rule gave
\emph{different} labels, then checked which one the model's classifications followed. This was
the test that distinguished ``the model failed to articulate'' from ``the model learned a
different rule.''

\begin{promptbox}{Step 1 prompt (classify, no reasoning)}
\ttfamily\scriptsize
\textit{system:} You are a careful classifier\dots\ Infer the rule from the examples and
apply it to the final input. Respond with exactly one word, either True or False\dots\\[2pt]
Here are labeled examples:\\
Input: "10 build lemon grape grape bright"  Label: True\\
Input: "cloud bear dog cook plum"  Label: False\\
\dots\ (\,$\sim$50 examples, balanced\,) \dots\\[2pt]
Now classify the next input. Respond with only True or False.\\
Input: "write house for cat slow fast 7"  Label:
\end{promptbox}

\begin{promptbox}{Step 2 prompt (articulate, no reasoning)}
\ttfamily\scriptsize
\textit{system:} \dots\ Respond with exactly one line\dots\ in this format:
RULE: <one sentence stating the rule>. Do not reason\dots\\[2pt]
Here are labeled examples: \dots\ (same examples) \dots\\
State the rule.
\end{promptbox}

\section{Results on Claude Opus 4.8}
\label{sec:claude}

\subsection{Which rules were learnable in context}
A sweep over my seventeen rules split them cleanly (Table~\ref{tab:classify}). It seemed that rules resting on a feature the model tracked as it read---a digit in the middle somewhere, the whole
string being lowercase, a trailing ``?'', the first word's part of speech, etc.---classified at
90--100\%. Rules that required counting (even word count, 54\%) or isolating a
position-specific letter (last letter is a vowel, 50\%) sat at 50:50.

I conducted three controls to check this. (i)~A \emph{zero-shot} baseline (no examples) put \emph{every} rule at
chance (42--57\%), so the competence was not prior knowledge but genuinely in-context learning.
(ii)~\emph{Scaling} \rulename{last\_char\_vowel} to 18/50/100/200 examples left it flat
at chance (54/50/48/52\%) while \rulename{all\_lowercase} stayed at 100\%. That means addingmore data did
not rescue a sub-symbolic rule. (iii)~Accuracies remained stable across three example sets, with
non-overlapping confidence intervals between learnable and unlearnable groups.

The reason for the split is, I believe, mechanical. The model reads text as tokens, of course, not
letter by letter. (This is the same limitation behind infamously miscounting the r's in ``strawberry''). So 
word-internal spelling was largely inaccessible. The split was \emph{not} meaning vs.\
surface---``ends with ?'' was a surface feature learned perfectly, because ``?'' is its
own token. I tried a clean \emph{salience gradient} experiment which appeared to confirm the mechanism: the \emph{same}
``is a vowel'' feature was learnable at 88\% for the \emph{first} letter of the string,
66\% at the start of the second word, and 50\% (chance) for the \emph{last} letter.

\subsection{Articulation}
I asked the model to state every rule that cleared 90\%, over six independent example
sets. The five \emph{meaning-based} rules stated the intended rule on all 6/6 attempts,
even with no reasoning (Table~\ref{tab:artic}). The two \emph{letter-based} rules almost
never did: \rulename{first\_letter\_vowel} stated it on 2/6 attempts and
\rulename{hunt\_first\_letter\_ae} on 1/6. 

Plotted as classification accuracy on
$x$, articulation rate on $y$, this looked like my money quadrant: two rules sat in the
bottom-right, ``classifies but cannot articulate.''

However, that reading was wrong, as the next
experiment showed: when \rulename{first\_letter\_vowel} failed to state the rule, it did not
flounder. It confidently articulated a \emph{different, simpler} rule, ``the first word is a
function word (in, at, on, of, and, by, is, a),'' which happened to fit the training
labels because in my vocabulary vowel-initial words were mostly function words.

\subsection{The counterfactual test: substitutes, not gaps}
\label{sec:cf}
High Step-1 accuracy does not show, at least on its own, that the model had learned the rule I
intended. A handful of examples don't pin down a single rule. Many
different rules can label the same examples the same way.

For \rulename{first\_letter\_vowel} two rules fit almost equally well: The
\emph{intended} rule was ``the first letter is a vowel.'' A simpler \emph{substitute} was
``the first word is a short function word'' (like \emph{in}, \emph{at}, \emph{on},
\emph{of}, \emph{and}, \emph{is}). The two agreed on the training data only by coincidence: the words in my generator that happened to start with a vowel were almost
all function words, so ``starts with a vowel'' and ``is a function word'' produced the same
label on nearly every training example. Ordinary accuracy therefore could not tell them
apart. Both rules score $\sim$88\%.

To separate them I built \emph{minimal pairs}: inputs on which the two rules \emph{disagree},
so the model's answer reveals which one it is actually using. Two kinds of input do this.
(i)~A vowel-initial \emph{content} word that is not a function word---and, crucially, a
\emph{novel} word the model never saw in training, such as ``orange'' or ``eagle.'' Here
the intended rule would label the input \textsf{True} (it starts with a vowel) but the
substitute \textsf{False} (it is not a function word). (ii)~A function word that starts
with a \emph{consonant}, such as ``by'' or ``the.'' Here the intended rule would label it
\textsf{False} but the substitute \textsf{True}. On each such input the two hypotheses
predict opposite labels, so whichever label the model gave identified the rule it had
really learned.

Across 300 of these disagreeing inputs the model followed the \emph{substitute} 63\% of the
time and the intended letter rule only 37\% (95\% CI 32--43\%; Fig.~\ref{fig:faith}). The
rule the model had \emph{articulated} (``the first word is a function word'') was thus also
the rule that actually \emph{governed} its behaviour. Its articulation was faithful to what
it did. Only my \emph{intent} diverged from both. A memorisation probe confirmed the
mechanism. Words beginning with vowels that the model \emph{had} seen in training were labelled
\textsf{True} 48\% of the time, but genuinely novel words beginning with vowels only 29\%---a
19-point gap showing the model had memorised a few specific training words rather than
learning the general ``starts with a vowel'' rule.

This was \emph{underdetermination}. A finite set of examples is consistent with many rules,
and the model seemingly learned, used, and reported the easier of the two it could not
distinguish. Figure~\ref{fig:hero} makes the point across all five rules that classified at
$\ge$90\%. Ordinary in-distribution accuracy (grey) was high for every one and revealed
\emph{nothing} about which rule had been learned; only the counterfactual test---\emph{genuineness},
the fraction of disagreeing inputs on which behaviour followed the \emph{intended} rule
(coloured)---told them apart. It cleanly separated the three genuine rules
(\rulename{first\_word\_verb} 97\%, \rulename{is\_question} 97\%, conjunction 100\%) from
the two substitutes (\rulename{first\_letter\_vowel} 37\%, \rulename{hunt\_first\_letter\_ae}
51\%, dropping to 12\% on the sharpest test, whether it applied the rule to unfamiliar
words). What decided which group a rule fell into was whether its intended feature was one
the model could represent in terms of meaning, not how hard the rule was in the abstract.

\subsection{The model knew the concept; the substitute was controllable}
I conducted two further experiments to rule out the boring explanation and show the phenomenon was not a fixed
limit. First, an \emph{isolation probe}: asked directly ``Does `orange' start with a
vowel? Yes/No'' for 24 words---including the exact novel words it labelled \textsf{False}
and function words it labelled \textsf{True}---the model was correct 24/24 (100\%). So the
failure was not one of knowledge. Second, I could switch the substitute off entirely (Fig.~\ref{fig:control}), from either
direction. \emph{Decorrelating} the training examples, so the function-word shortcut no
longer fit, made behaviour follow the intended rule 95\% of the time, up from 39\%. And
simply \emph{letting the model reason} before it answered, on the very same ordinary
examples, did much the same: 91\%, up from 38\%. So the model had the real rule available
all along. Which rule it deployed seemed to depend on how underdetermined the examples
were, and whether it was allowed to reason, not on any inability of its own.

One subtlety is worth flagging. Chain-of-thought can mislead here when it is used only to
\emph{articulate} the rule, because the model's no-reasoning \emph{behaviour} may still be
following the substitute. The mismatch is then between reasoned articulation and unreasoned
behaviour, not a fixed property of the model.

\section{Open-weights replication: reading the rules off the activations}
\label{sec:gemma}
Everything so far was behavioural, and with a closed model like Claude that is all we get
to see. So I repeated the load-bearing experiments on Gemma~4 31B, an open-weights model
whose residual stream I can read directly. \rulename{first\_letter\_vowel} was learnable
there too, at 90\% in-distribution (alongside \rulename{all\_lowercase} 85\%,
\rulename{is\_question} 85\%, and the conjunction 80\%).

\paragraph{Behaviour: a blend, not a collapse.} I re-ran the \S\ref{sec:cf} counterfactual
as a balanced $2{\times}2$ (vowel/consonant first letter $\times$ content/function first
word). On the discriminating cells, Gemma matched the pure letter rule 56\% of the time.
Reading the edges of the table, switching the first letter to a vowel added about 39 points
of \textsf{True}, and switching the first word to a function word added about 27; the two
effects roughly added. So Gemma seemed to use \emph{both} rules at once, with the intended
letter rule the slightly \emph{stronger} driver. That is a real difference from Claude,
which collapsed onto the substitute.

\paragraph{Representation: both rules are written down.} I then trained a simple
logistic-regression probe on the residual stream at the answer position, the token right
after \texttt{Label:}. I split train and test \emph{by word}, so the probe had to read the
concept rather than memorise which words it had seen, and I used the decorrelated
$2{\times}2$ so that each cue was independent of the other. As a check that the pipeline
worked at all, a probe reading ``first letter is a vowel'' off a bare word reached 99\% on
held-out words. At the decision point itself (Fig.~\ref{fig:probe}), both cues were
linearly decodable well above chance: the intended letter rule peaked at 86\%, the
function-word substitute at 96\%.

So the underdetermination was not just an inference from behaviour. It was a measured fact
about the representation: both competing rules coexisted in the activations, and the data
never forced a choice between them. Two things temper this. The substitute was the more
decodable of the two, and stayed so right to the final layer (78\% vs.\ 64\%), even though
the counterfactual had found the letter the stronger behavioural driver. Decodability is
not the same as causal use, so I should not over-read the gap.

\section{Discussion}
The whole study points to one claim. In-context learning recovered the rule I intended
whenever that rule rested on a feature the model already represents in terms of
meaning---a word's part of speech, the punctuation that ends a sentence, the presence of a
digit. When the intended feature was instead sub-symbolic, like a particular letter inside
a word or a count, the model quietly substituted an easier feature that gave the same
answers on the training data, and then both used and reported that substitute. So the
apparent ``articulation failure'' was not the model failing to introspect, and it was not
dishonesty. It was underdetermination: the model had learned a different rule from the one
I had in mind, and told me so faithfully. Only a counterfactual or out-of-distribution test
tells the two apart.

The practical lesson is a methodological one: \emph{in-distribution accuracy tells you
almost nothing about which rule a model has actually learned.} A rule can be classified at
100\% and articulated fluently while the rule the model articulates, the rule that drives
its behaviour, and the rule I intended are three different things. Grading an articulation
against the intended rule alone would wrongly mark a faithful substitute as a ``cannot
articulate'' case. Only the counterfactual test separates them.

This also speaks to the dishonesty worry in the brief, and my answer is the more
interesting negative. The model was not hiding a rule it was using. It used the substitute
and it reported the substitute. It plainly knew the underlying letter concept---it scored
100\% when I asked in isolation---but that knowledge simply was not recruited when it
learned the rule from examples. Reasoning, or decorrelated data, recruited it.

\paragraph{Limitations.} The substitute finding rested on two letter rules, both about the
\emph{first} letter; a substitute from a different sub-symbolic feature would broaden the
claim. \rulename{first\_word\_verb} was near the 90\% boundary (it dipped to 80\% on one
seed). Articulations were graded by eye; a validated automatic grader would be needed to
scale to many rules. The Gemma probe rested on modest
data ($\pm$2--5\%), and crucially it showed what was linearly \emph{present}, not what was
causally \emph{used}. I'd love to steer along the probe
direction and check whether the answer moves.
